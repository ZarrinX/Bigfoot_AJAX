Frequently Asked Questions


Since our recent change in default hosting nameservers, determining which nameservers to use can be confusing. ATS has developed a new tool for determining which one of our nameservers should be used for a hosted domain. This tool queries all of our nameservers for the SOA record of a domain. Contained is the SOA record is the "Primary Name Server" field which indentifies the nameserver that has authority over a domain.

Q: When would I need to use this tool?
A: The NS dig tool should be used when a hosted domain does not resolve. A good example would be when a domain is using our nameservers, but dnsstuff.com shows no records for the domain. It's possible the domain is not using the server that has the zone file.

Q: What does AUTHORITATIVE, NON AUTHORITATIVE and NO ANSWER mean?
A: AUTHORITATIVE- These are the primary nameservers listed for the domain. This is where the zone file lives, and where the domain is hosted. The domain should be pointed at these servers.

NON AUTHORITATIVE- These server contain duplicate records of the primary nameserver. While the domain could be placed here, it is always good practice to use the primary nameservers.

NO ANSWER- This server contains no zone information about the domain and should not be used.

Note: This list does not display the parked nameservers.

Q: The domain is hosted here, but the only records that were found are on a parked server. Is something wrong?
A: No. When hosting is setup for a domain that is on our parked server, the hosting servers do not create a separate zone file on the hosting servers. If the doman is hosted here it can continue to use the parked servers, but it should be moved to the default hosting nameservers if there is a problem. This action will create a new zone file on the hosting server.

Q: What if the domain is using 3rd party nameservers? Will the NS audit tool still work?
A: Yes. The dig tool does not depend on the domain when running queries. It will scan our servers for the domain records regardless of what nameservers are currently used by the domain.

Q: I have several aliased domains on my hosting account. Would these use the same nameservers as the primary domain?
A: Not always. Aliased domains follow the same rules as hosting accounts, it depends on when it was set up. The dig tool can be used to determine this.
